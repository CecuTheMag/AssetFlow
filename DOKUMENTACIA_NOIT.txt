АСЕТФЛОУ (ASSETFLOW) - ПРОФЕСИОНАЛНА СИСТЕМА ЗА УПРАВЛЕНИЕ НА ИНВЕНТАР
Документация за Национален есенен турнир по информационни технологии "Джон Атанасов"
Категория: Софтуерни приложения (10-ти клас)

РЕЗЮМЕ

1. ЦЕЛИ (предназначение, кратък анализ на потребностите и на съществуващите решения)

ПРЕДНАЗНАЧЕНИЕ:
AssetFlow е професионална система за управление на инвентар, специално проектирана за образователни институции. Системата автоматизира процесите по заявяване, одобряване и проследяване на оборудване, осигурявайки ефективно управление на ресурсите.

АНАЛИЗ НА ПОТРЕБНОСТИТЕ:
• Образователните институции се сблъскват с предизвикателства при управлението на хиляди единици оборудване
• Липса на централизирана система за проследяване на наличност и местоположение
• Неефективни процеси за заявяване и одобряване на оборудване
• Затруднения при планиране на учебни дейности поради неясна наличност на ресурси
• Липса на аналитични данни за оптимизация на използването

АНАЛИЗ НА СЪЩЕСТВУВАЩИТЕ РЕШЕНИЯ:
• Традиционните системи са скъпи (над 50,000 лв.) и сложни за внедряване
• Повечето решения не са адаптирани за образователната среда
• Липсват интеграции с учебни планове и програми
• Ограничена функционалност за различни роли (ученици, учители, администратори)
• Слаба мобилна поддръжка и модерен потребителски интерфейс

4. ЛОГИЧЕСКО И ФУНКЦИОНАЛНО ОПИСАНИЕ НА РЕШЕНИЕТО

СИСТЕМНА АРХИТЕКТУРА:
AssetFlow следва микросервисна архитектура с ясно разделение на отговорностите:

FRONTEND СЛОЙ (React SPA):
• Компонент за автентикация и управление на сесии
• Dashboard с real-time статистики и нотификации
• Модул за управление на оборудване с CRUD операции
• Система за заявки с workflow управление
• Административен панел за системни настройки
• Мобилно-адаптивен дизайн за всички устройства

API СЛОЙ (Node.js/Express):
• Authentication Service - JWT базирана автентикация
• Equipment Service - управление на инвентар
• Request Service - обработка на заявки и одобрения
• User Service - управление на потребители и роли
• Notification Service - real-time нотификации
• Analytics Service - генериране на отчети и статистики

ДАННИ И ПЕРСИСТЕНТНОСТ:
• PostgreSQL - основна база данни с ACID гаранции
• Redis - кеш за сесии и често използвани данни
• File Storage - сигурно съхранение на документи и изображения

ИНФРАСТРУКТУРА:
• Docker контейнери за всички сервиси
• Kubernetes за оркестрация и автоматично скалиране
• Nginx като reverse proxy и load balancer
• Prometheus + Grafana за мониторинг

ВЗАИМОДЕЙСТВИЯ МЕЖДУ МОДУЛИТЕ:
1. Потребителят се автентикира чрез Authentication Service
2. Frontend получава JWT токен и го използва за всички заявки
3. API Gateway рутира заявките към съответните микросервиси
4. Всеки сервис валидира правата и обработва заявката
5. Данните се кешират в Redis за по-бърз достъп
6. Нотификациите се изпращат real-time чрез WebSocket

5. РЕАЛИЗАЦИЯ

ОБОСНОВКА ЗА ТЕХНОЛОГИЧНИТЕ СРЕДСТВА:

FRONTEND ТЕХНОЛОГИИ:
• React 18 - модерна библиотека с отлична производителност и екосистема
• Vite - бърз build tool с hot module replacement
• Material-UI - професионални компоненти с accessibility поддръжка
• Axios - HTTP клиент с interceptors за error handling
• React Router - client-side routing за SPA функционалност

BACKEND ТЕХНОЛОГИИ:
• Node.js - асинхронна архитектура, подходяща за I/O интензивни операции
• Express.js - минималистичен framework с богата екосистема
• JWT - стандартизиран начин за сигурна автентикация
• bcryptjs - криптографски сигурно хеширане на пароли
• Joi - валидация на входни данни за предотвратяване на атаки

БАЗА ДАННИ:
• PostgreSQL 15 - ACID съвместимост, сложни заявки, JSON поддръжка
• Sequelize ORM - защита срещу SQL injection, миграции
• Connection pooling - оптимизация на връзките към базата данни

ИНФРАСТРУКТУРА:
• Docker - контейнеризация за консистентна среда
• Kubernetes - автоматично скалиране и self-healing
• Redis - in-memory кеш за сесии и често използвани данни
• Nginx - reverse proxy, load balancing, SSL termination

АЛГОРИТМИ И ПОДХОДИ:
• JWT с refresh tokens за сигурна автентикация
• Role-based access control (RBAC) за управление на права
• Optimistic locking за предотвратяване на race conditions
• Caching strategies с TTL за оптимизация на производителността
• Rate limiting за защита срещу DDoS атаки

ИЗПОЛЗВАНА ЛИТЕРАТУРА:
• "Node.js Design Patterns" - Mario Casciaro
• "React Design Patterns and Best Practices" - Michele Bertoli
• "PostgreSQL: Up and Running" - Regina Obe
• "Microservices Patterns" - Chris Richardson
• MDN Web Docs за web стандарти и best practices

6. ОПИСАНИЕ НА ПРИЛОЖЕНИЕТО

СИСТЕМНИ ИЗИСКВАНИЯ:
• Операционна система: Windows 10+, macOS 10.15+, Linux Ubuntu 18.04+
• RAM: минимум 8GB (препоръчително 16GB)
• Свободно дисково пространство: 10GB
• Docker Desktop 4.0+ и Docker Compose
• Интернет връзка за изтегляне на dependencies

ИНСТАЛАЦИЯ И СТАРТИРАНЕ:

ВАРИАНТ 1 - DOCKER (ПРЕПОРЪЧИТЕЛНО):
1. Селектиране на папката:

    cd */AssetFlow

2. Стартиране на приложението:

    docker-compose up --build

ВАРИАНТ 2 - ЛОКАЛНА РАЗРАБОТКА:
1. Инсталиране на Node.js 18+ и PostgreSQL 15+
2. Инсталиране на dependencies:
   npm install (в backend и frontend папките)
3. Конфигуриране на .env файлове
4. Стартиране на базата данни и Redis
5. Стартиране на backend: npm run dev
6. Стартиране на frontend: npm run dev
